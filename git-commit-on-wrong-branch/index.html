Git commit on wrong branch

This can be fixed in 3 steps

1. Create a patch with what you’ve committed

2. Apply the patch on the correct branch

3. Delete the commit from the wrong branch.

1. Create a patch with what you’ve committed
git format-patch -1 &lt;sha&gt; --stdout &gt; my_commit_file.patch
where sha is the commit’s sha , to find it you can use git log

2. Apply the patch on the correct branch
git checkout &lt;branch_name&gt; git am --signoff &lt; my_commit_file.patch

3. Delete the commit from the wrong branch.
git checkout &lt;branch_name&gt;
git reset --hard &lt;sha1-commit-id&gt;
If you already pushed it, you will need to do a force push to get rid of it…
git push origin HEAD --force
