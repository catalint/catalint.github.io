Windows + Git + SSH - A tutorial

Have you ever wanted to use Git over SSH on Windows? Of course you have. After searching the internets, reading over 10 similar tutorials and banging my head on the keyboard for about four hours i managed to get it working.

Difficulty rating: Easy/Medium.

This tutorial assumes you are on a clean machine and you don’t have Git installed or you have another version of Git. Feel free to skip ahead if you have the current version.

  1. Download the latest Git release from here and install it.

  2. Open Git Bash and perform the initial Git setup by running the following commands:

    1. git config --global user.name "&lt;Your Name&gt;"
    2. git config --global user.email "&lt;Your E-mail&gt;"

  3. Go to the .ssh directory which is located in your home directory. In Windows Vista and up (which you should be using in 2013, btw) this directory is in C:\Users\<Your Name>.ssh. From the Git Bash you can get get there using “cd ~/.ssh”. If it does not exist create it now.

  4. List the files in this directory using “ls”. If you see two files named id_rsa and id_rsa.pub you can skip the next step.

  5. You have to create a private/public RSA key pair. To do this, run the command “ssh-keygen -t rsa“.

    1. It will ask for the directory and file name for your new RSA key. Hit Enter to use the defaults.
    2. Then it will ask for a passphrase. It’s not recommended to leave this blank unless you need this key for an automated script. Enter your passphrase (or leave it blank at your own peril) and hit Enter.
    3. Confirm your passphrase.
    4. List the contents of the directory again, marvel at the sight of your new RSA keys and give yourself a pat on the back.

  6. Now you have to tell Windows where to find Git and your RSA keys:

    1. Go to Control Panel -> System -> Advanced System Settings -> Environment Variables.
    2. In the user variables section (the one at the top) you have to add a new variable named HOME with the value %USERPROFILE%. DO NOT add this to the global variables section or you too will be banging your head on the keyboard for four hours asking yourself why Windows or your IDE can’t communicate with the Git server.
    3. In the system variables section you have to edit the Path variable and add the path to the Git bin directory which contains the Git executable and other useful tools. Typically (on a 64-bit system) this is “C:\Program Files (x86)\Git\bin“.
    4. Click on all the OK buttons to save your settings.
    5. Now you can use git, ssh, cat and all the awesome tools that come with Git in any command prompt or powershell without having to start Git Bash every time AND Window knows where to find your RSA keys.

  7. Go to your Git server administration panel/console/whatever and add the contents of the id_rsa.pub file to the SSH Keys section, authorized_keys file or wherever you can add RSA keys for automatic SSH authentication. Do not copy any spaces when you copy the RSA key or it won’t be recognized.

  8. Feel free to git pull or git push over SSH as your heart desires and don’t forget to give yourself another pat on the back for a job well done.
